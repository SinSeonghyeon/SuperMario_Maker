<기본적인 용어>
 - 쪼개져 있는 장식품의 경우
	- L = Left
	- R = Right
	- Top - 위쪽
	- Body - 내부를 채워주는 느낌
	- U = Up
	- D = Down
 - 두개가 합쳐져 있으면 ex)LD => 왼쪽 아랫부분. 이런 느낌으로 이해하면 될 것 같습니다.
 - 기본적으로 투명키는 (0, 255, 0)입니다.

<BackGround>
 - 충돌 구현 없이 가장 먼저 그려주는 비트맵들
 - 배경 지우기 용으로 BGColor를 넣음 해당 색의 RGB값 => (92, 148, 252)
	- 시작할 때 맵 전체를 해당 색으로 그려주면 될듯 합니다
 - 나머지는 현재는 쪼개진 상태로 뒀습니다. 타일에디터가 완성되면 추가기능을 넣어서
   배경화면 메이커를 만들어도 괜찮을 것 같아서 쪼개진 상태로 뒀습니다.
 - 여기 있는 비트맵들은 충돌 계산을 하지 않아도 되는 것 들입니다.
	- 화면상 뒤쪽에 구현되는 것들

<Block, Item, Enemies>
 - 충돌 구현이 필요한 비트맵들.
 - 블록은 충돌했을 때 길을 막는 느낌, Item은 트리거, Enemies는 충돌하면 사망처리
 - 블록 중에 파편은 넣어는 봤는데 사용할 수 있을진 모르겠습니다..
 - 기본적으로 애니메이션을 가정하여 애니메이션이 있는 것은 0, 1, 2와 같은 식으로 처리했습니다.
